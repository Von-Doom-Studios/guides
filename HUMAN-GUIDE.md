## How to Talk to Your Agents

**Just message the right channel.** Each agent reads your message and responds directly.

**Tips:**
- Be specific about what you want. "Make content for Floki" → "Write 3 Instagram captions for the hanger launch, casual tone, focus on the color variety."
- You don't need to explain background — the agents have memory of what's been built.
- If one agent needs another to do something, you relay it. Agents can't talk to each other directly (by design).

**Example workflow:**
> You → #marketing: "I need launch content for the Floki hangers — Instagram, TikTok, X. Aim for next week."  
> Marketing responds with a content calendar.  
> Marketing → you: "I need product photos from Creative — can you ask them for 3 lifestyle shots?"  
> You → #creative: "Marketing needs 3 lifestyle shots for the hanger launch."

## Communication Rules

### What agents will do automatically
- Follow up on open threads without being asked
- Flag problems before they escalate
- Update their memory files at the end of every session

### What they won't do
- Email anyone externally without you telling them to (Ops relays external email)
- Push code to production without your awareness
- Spend credits analyzing large batches of images without asking first

### Cross-agent coordination
Agents work in their own lanes. When coordination is needed, you're the relay — just copy-paste or forward the message. This is intentional: it keeps you in control of what moves between departments.

## Agent Memory — How It Works

Sessions wipe every night. The agents remember what they've been told through memory files they write themselves. If something important isn't in the memory file, it's gone the next session.

**This means:**
- Always give corrections and decisions directly in the channel (don't assume they remember from weeks ago)
- If something was decided and the agent doesn't seem to know it, it may not have been written to memory — just tell them again
- Memory files live in GitHub and on the host machine
